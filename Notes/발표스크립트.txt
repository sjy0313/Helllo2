4페이지 : 지방소멸이란? 누군가가 살아왔고 현재도 생활터전이 되는 지역과 마을이 사라진다는 것
저출산 고령화와 대도시로의 인구 이동으로 인해 발생하는 지방의 과소지역화 내지는 무거주화 현상
일본 창성회의 의장 출신인 마스다 히로야가 처음 사용
5페이지 : 왜 지방 소멸인가? 사회에서 지역소멸 및 지방소멸 문제를 심각하게 인식 중,
이에, 지역 소멸위기 해소에 도움이 되고자 함
대한민국 지도에서 대한민국 인구가 5100만 이지만 서울 지역에 약 천만
경기도 지역에 약 1400만명의 인구가 살고 있다는 것을 표기해서 좁은 지역에 인구 절반이
몰려 산다는 것을 강조
6페이지 : 스크랩한 각종 기사들을 바탕으로 사회적으로도 큰 이슈가 되고 있음을 강조
7페이지 : 분석 대상은 전국입니다 기존에는 충청권 도시들로만 분석을 진행할 예정이었지만
분석의 범위가 좁다보니 정확도가 떨어지는 단점이 있었습니다
이에 분석의 신뢰도와 정확성을 높이기 위해 분석 대상을 전국으로 넓혔습니다

10페이지 : 3.개발 과정 데이터 수집 
11-12페이지 : (1) 데이터 수집
수집 방법 : 통계청(KOSIS)에서 원하는 자료 직접 수집
변수 선정 : 정윤이가 보내준 변수 목록 사진 + 대기질
선정 이유 : 한국지방정부학회의 한국 지방소멸 요인과 극복 방안에 관한 연구:머신러닝 방법을 통한 탐색
논문을 참고하여 변수 선정
데이터 범위 : 2015년을 시작 기점으로 잡고 최소 5년치 데이터 수집
13-14페이지 : (2) 데이터 전처리
결측치 확인 및 다중공선성 확인을 거친 후 필요시
Minmax,robust 스케일링 방식 활용
15-17페이지 : (3) 모델 구축
부스팅, Gradient Boosting Maching(GBM)과 XGBoost, Random forest
모델을 활용하여 성능 향상을 도모할 예정
18-19페이지 : (4) 향후 방향성
위의 과정을 거쳐서 나온 개선된 소멸위험지수를 바탕으로 대한민국 지도에 시각화
색깔이 진할 수록 소멸 위험도가 높고 색깔이 옅을수록 소멸 위험도가 낮은 지역
20-21페이지 : (5) DB설계 및 구축
이전 프로젝트에서 적재된 인구이동데이터를 활용
스파이더와 엑셀을 이용해서 새로 수집한 데이터들을
이미 적재된 인구이동데이터 형식에 맞춰서 전처리 후 DB에 데이터 적재
DBMS를 머신러닝 모델구축에 사용해야 하는 이유 -> 3차 미니프로젝트 발표 ppt 내용 가져오기
